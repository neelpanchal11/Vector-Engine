# Vector Engine v0.3.0 (Draft)

## Release Positioning

v0.3.0 is a stability-oriented integration release. It focuses on adoption and reproducibility workflows on top of the core feature surfaces introduced in v0.2.0-alpha.

## Highlights

- Public-ready demo workflow under `demo_repo_template/`:
  - semantic retrieval demo
  - item similarity demo
  - benchmark demo with bruteforce vs faiss_flat comparison
  - quickstart verification script (`scripts/verify_quickstart.py`)
- Stable integration guidance:
  - `docs/integration_guides.md`
  - `docs/reproducibility.md`
- Reproducibility-first command workflow and artifact policy:
  - baseline report
  - real-corpus evaluation report
  - repeated stability runs
  - exact-equivalence benchmark checks

## Migration from v0.2.0-alpha

- No breaking API changes are introduced in core `vector_engine` entry points.
- Existing v0.2.0-alpha workflows continue to run.
- Documentation language now positions the project as v0.3.0 stable.
- Integration and release workflows are now organized into explicit guides and publish/private artifact policy.

## Validation Evidence Snapshot

- Real-corpus style 3-run results remain stable (see `artifacts/real_corpus_runs/`).
- Faiss Flat exact-equivalence checks pass overlap gating (`overlap_vs_bruteforce >= 0.99` target, observed `1.0` in recorded runs).
- 200-run stability study generated with summary statistics and plot:
  - `artifacts/testing_runs/stability_runs_bruteforce_200.jsonl`
  - `artifacts/testing_runs/stability_summary_bruteforce_200.json`
  - `artifacts/testing_runs/stability_plot_p95_qps.png`
- Stability notebook:
  - `notebooks/04_stability_runs.ipynb`

## Known Limits

- Real private-corpus embeddings and raw IDs should remain outside the public repo.
- Published benchmark numbers are configuration-dependent; include hardware/runtime notes with external reports.
- Faiss benchmark paths require local Faiss installation (`faiss-cpu`).

## Release Checklist

1. Run tests: `python3 -m pytest -q`
2. Generate baseline: `python3 scripts/rag_baseline.py --output-dir artifacts --k 3`
3. Run real-corpus evaluation: `python3 scripts/rag_real_corpus_eval.py --embeddings ... --query-embeddings ... --ids ... --ground-truth ...`
4. Run stability workflow: `python3 scripts/stability_runs.py --embeddings ... --query-embeddings ... --ids ... --ground-truth ... --backend bruteforce --run-count 200 --output-dir artifacts/testing_runs`
5. Run exact benchmark: `python3 benchmarks/compare_bruteforce_vs_faiss.py --mode exact --min-flat-overlap 0.99 --output artifacts/faiss_equivalence/run_1.json`
6. Verify docs links and artifact policy language in `README.md`.
7. Tag and release:
   - `git tag -a v0.3.0 -m "Vector Engine v0.3.0"`
   - `git push origin v0.3.0`
