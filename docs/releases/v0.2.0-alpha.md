# Vector Engine v0.2.0-alpha (Draft)

## Highlights

- Expanded clustering API with richer outputs:
  - labels
  - centers
  - inertia
  - iteration count
- Hard-negative mining strategies:
  - `top1`
  - `topk_sample`
  - `distance_band`
  - plus exclusion support (`exclude_ids`, `exclude_mask`)
- Retrieval evaluation enhancements:
  - `retrieval_report_detailed` (summary + per-query)
  - `batch_metrics_summary` for multi-run aggregation
- Real-corpus evaluation script:
  - `scripts/rag_real_corpus_eval.py`
  - supports retrieval thresholds and latency gates
- Public demo template included in `demo_repo_template/`

## Validation Snapshot

- RAG baseline artifact generation validated.
- Real-corpus style 3-run reports generated in `artifacts/real_corpus_runs/`:
  - quality stable across runs (`recall@1/3/6 = 1.0`, `ndcg@1/3/6 = 1.0`)
  - p95 latency envelope: `0.0376-0.0717 ms`
  - thresholds pass in all runs (`recall >= 0.75`, `ndcg >= 0.70`, `p95 <= 120 ms`)
- Faiss Flat exact-equivalence checks generated in `artifacts/faiss_equivalence/`:
  - `overlap_vs_bruteforce = 1.0` in all 3 runs with `--min-flat-overlap 0.99`
  - p95 latency ranges:
    - bruteforce: `29.99-37.63 ms`
    - faiss_flat: `4.17-15.03 ms`
- 200-run stability study generated:
  - `artifacts/testing_runs/stability_runs_bruteforce_200.jsonl`
  - `artifacts/testing_runs/stability_summary_bruteforce_200.json`
  - p95 mean `0.0255 ms` (95% interval `0.0203-0.0547 ms`)
  - qps mean `188,097` (95% interval `117,499-214,111`)
- Stability analysis notebook: `notebooks/04_stability_runs.ipynb`.

## Known Limitations

- Real-corpus evaluation requires user-provided embeddings/ground-truth files.
- Public demo template is included locally; publish as separate repo for outreach.
- Mock/public-safe corpus numbers are not a substitute for production-scale private corpus benchmarks.

## Release Checklist

1. Run `python3 -m pytest -q`
2. Run `python3 scripts/rag_baseline.py --output-dir artifacts --k 3`
3. Run exact equivalence:
   - `python3 benchmarks/compare_bruteforce_vs_faiss.py --mode exact --min-flat-overlap 0.99 --output artifacts/benchmark_exact.json`
4. Run real corpus eval:
   - `python3 scripts/rag_real_corpus_eval.py --embeddings ... --query-embeddings ... --ids ... --ground-truth ...`
5. Run stability study:
   - `python3 scripts/stability_runs.py --embeddings ... --query-embeddings ... --ids ... --ground-truth ... --backend bruteforce --run-count 200 --output-dir artifacts/testing_runs`
6. Update README benchmark and quality numbers
7. Tag and release:
   - `git tag -a v0.2.0-alpha -m "Vector Engine v0.2.0-alpha"`
   - `git push origin v0.2.0-alpha`
